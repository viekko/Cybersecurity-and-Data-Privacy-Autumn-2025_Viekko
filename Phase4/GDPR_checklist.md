# GDPR-tarkistuslista – Varausjärjestelmä (Phase 4)

| GDPR-vaatimus | Täyttyy | Huomiot |
|--------------|--------|---------|
| Lainmukaisuus, kohtuullisuus ja läpinäkyvyys | Kyllä | Käyttäjille on saatavilla tietosuojaseloste |
| Käyttötarkoitussidonnaisuus | Kyllä | Henkilötietoja käytetään vain palvelun toteuttamiseen |
| Tietojen minimointi | Kyllä | Kerätään vain sähköposti, ikä ja varaustiedot |
| Tietojen paikkansapitävyys | Osittain | Käyttäjä syöttää tiedot itse |
| Säilytyksen rajoittaminen | Osittain | Tietoja säilytetään niin kauan kuin käyttäjätili on olemassa |
| Eheys ja luottamuksellisuus | Osittain | Käyttöoikeudet rajoitettu, mutta salauksesta ei tietoa |
| Käsittelyn oikeusperuste määritelty | Kyllä | Sopimus (GDPR 6 artikla 1(b)) |
| Privacy by Design -periaate | Kyllä | Vain tarpeelliset tiedot kerätään |
| Oikeus tarkastaa tiedot | Osittain | Ei automaattista toimintoa |
| Oikeus tietojen oikaisuun | Osittain | Käyttäjä ei voi muokata kaikkia tietoja itse |
| Oikeus tietojen poistoon | Osittain | Ylläpitäjä voi poistaa käyttäjän |
| Oikeus saada tietoa | Kyllä | Tietosuojaseloste saatavilla |
| Suostumus vaaditaan | Ei | Käsittely perustuu sopimukseen |
| Evästekäytäntö saatavilla | Kyllä | Käytössä vain istuntoevästeet |
| Tietojen luovutus kolmansille osapuolille | Ei | Tietoja ei jaeta |
| Erityisiä henkilötietoryhmiä käsitellään | Ei | Ei arkaluonteisia tietoja |

## Yhteenveto
Varausjärjestelmä noudattaa GDPR:n perusperiaatteita.  
Joidenkin rekisteröidyn oikeuksien toteutus vaatii ylläpitäjän toimenpiteitä, mutta tämä on hyväksyttävää yksinkertaisessa järjestelmässä.
